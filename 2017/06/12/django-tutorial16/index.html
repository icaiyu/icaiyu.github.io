Django新手教程16，用户注册页面

发表于 2017-06-11 | 分类于 django_tutorial | 阅读次数
字数统计 | 阅读时长

非常遗憾，注册需要用到的讲解员，django是不内置这样的讲解员的，你必须自己在讲解员名单里面添加，所以，如果你只是小范围使用的话，就在管理页面手动添加用户就好了，不要什么注册系统，本章就照顾一下有雄心壮志的同学，他们肯定是需要制作面向广大群众的系统的，所以会员注册肯定要搞起来。
我们这里讲解如何制作用户的注册页面，我们先理一下逻辑，一个新用户登陆我们的页面，他首先看到的应该的登录页面，如果他没有账号，那么他需要点击一个注册的按钮，前往注册页面，所以，跟往常一样，我们需要三样东西：

  1. 设置URL，这个不必多说，
  2. 聘请讲解员，他负责将注册页面的用户资料存进数据库。
  3. 制作html文件，这个是用户注册时看到的页面，这上面应该有一个HTML表单用来收集用户的信息，比如用户名和密码。
    所以，我们首先需要打开myCloud目录下的urls.py文件

我们的讲解员的逻辑应该是怎么样的呢？首先他要对request的方式做一下判断，如果是get，那么说明游客只是跳转到了注册页面，准备填写资料，因此，我们应该渲染一个HTML表单嵌入到signup.html，而这个表单django已经帮我们设计好了，我们就不用自己写以下这些乱七八糟的东西

我们只需要
from django.contrib.auth.forms import UserCreationForm
然后
form = UserCreationForm()
这个form就包含了用户注册需要的东西，比如用户名，密码之类
我们只要把这个form当做是讲解语嵌入到signup.html里面。
return render(request, ‘registration/signup.html’, {‘form_jiangjieyu’: form})
如果request的方式是post，说明用户已经填好资料了，那么我们首先验证资料是否有效。
if form.is_valid():
有效的话，我们就使用
form.save()
这时就把新注册的用户添加到用户库里面去了，然后我们希望，接下来就以这名新用户的名义转到主页上去，所以，我们需要用这个用户名和密码登录一下
username = form.cleaned_data.get(‘username’)
userpassword = form.cleaned_data.get(‘password1’)
user = authenticate(username=username, password=userpassword)
login(request, user)
return redirect(‘/‘)
