django-tutorial5, IDE

发表于 2017-06-11 | 分类于 django_tutorial | 阅读次数
字数统计 | 阅读时长

聊完URL，咱们总算可以开始django项目了吧，NO，NO，NO，我早就说了，我写的是一本高大上的教程，肯定不能按套路出牌，咱们再扯点别的，比如IDE。
这年头，码农之间交流不讲点英文缩写啥的，都不好意思出门见人，这个IDE可是高频词汇，必须得掌握啊。
IDE，Integrated Development Environment
不少人认为IDE就是编写代码的工具，说白了，就是编辑器，其实还是有些差别的，IDE是开发项目的工具（我觉得这些概念意思意思就得了，也不要太较真，钻牛角尖），注意，我这里说的可以开发项目，而不是写代码，因为开发项目包含了写代码，调试，除虫，维护等一系列过程，编辑器只能帮助我们写代码，比如我们打开windows自带的田牌记事本，这就是一个典型的编辑器，我们可以敲入代码了，然后改个后缀名，就可以跑一下。这过程真繁琐，我们多希望记事本上有个叫运行的按钮，一按就可以运行代码多好，出啥错误可以跳出一个控制面板告诉我们错误在哪里，最好，还要有断点测试，单步测试等按钮，加上这些功能的编辑器，我们就可以称之为IDE了。愿望是美好的额，现实却无比残酷，人家田牌的记事本就不是给码农设计的，所以，我们得用别的IDE，高大上的那种，功能齐全的那种，俗话说，工欲善其事必先利其器，好的IDE能够提高我们的开发效率，尽快走向人生巅峰，那些鼓吹使用记事本写代码的人，你千万要小心，要么他真的是水平出臻化境，做到了人剑合一，拈花飞叶，不在乎手上拿的是什么器，要么就是装逼装到需要雷一下才清醒的，前一种，人家跟你就不是一个世界的，后一种咱们也别靠的太近小心被带偏节奏。
我当年用linux系统的时候，不知道听哪个家伙（可能是某大神或伪大神）鼓吹，vim是神器，结果折腾的我那个惨啊，vim这玩意，新手上路，别说翻车，车都上不去，前段日子。stack overflow达成了一项成就，帮助一百万新手正确退出vim（这里的退出指的是关闭编辑器，不是放弃使用的意思），是的，你没有看错，vim这东西想正确退出都是一场噩梦。折腾了好久，自认为也学了好多指令，就盼望着能够迅速提升开发效率呢，结果，没过几天一切都忘得干干净净。
所以，作为新手，就要有新手的觉悟，挑选合适的IDE至关重要，这里，我推荐的是jetbrain家的全家桶，不同于国内某些公司硬往别人电脑塞全家桶，这家人的全家桶几乎每一个都是收费的，这年头各种开源软件百花怒放，他们家的IDE敢收费，说明还是有两把刷子的。如果你是学生（有教育邮箱），你可以免费获得完整版的全家桶，如果不是，可以下载教育版，阉割了一些功能，不过依然非常强大。当然，如果我们只是开发python，就不需要下载全家桶，下载他们家的pycharm就可以了
https://www.jetbrains.com/pycharm-edu/download/#section=windows
当然，这IDE功能实在是太强大了，我们这里的项目用不到，所以，我们的教程还是基于sublime text3，到这里安装下载就好了
https://www.sublimetext.com/3

因为sublime确实好评如潮，有口皆碑，是久经考验的好战士。
咋一看名字，这只是一个编辑器，但是人家除了编辑功能强大到逆天，有很强大的插件（自行知乎sublime插件），还有那配色，界面也很好看，这年头，颜值才是生产力，用上了之后，欲罢不能（更多信息请百度或者知乎），sublime也支持直接运行python代码（它会帮忙调用系统的python编译器来运行），因此我觉得这个算得上是IDE。只是他的代码调测功能没有pycharm那么强大。他的 侧重点还是编辑器功能那一块。

顺便说一下，sublime text是不免费的，学生也不行，但是你可以下载使用，只不过会偶尔在你要保存文件的时候提示你一下，跟你要钱，这个频率不高，基本没啥困扰，跟国内免费手游乱放广告是不一样的。据说有学生写邮件说，希望能对学生打折，这样，大家愿意付点钱，不过sublime家拒绝了，就是这么傲娇，能怎么办呢，当然是原谅她啊，因为确实是神器啊。

此外，为什么我们选择调试功能稍弱的sublime来写项目呢，因为我们是要做网络服务啊，最后的调试工具应该是一个牛逼的浏览器，这里，我强烈建议大家使用火狐或者chrome。我们之后的讲解也会用到chrome，不过edge最近也颇有改头换面的意思，其实也挺好用的，chrome真的好占内存。

想不到吧，一个IDE的选择居然还有真么多门道。当然，除了我介绍的，python的IDE有很多选择，你可以随意挑选自己喜欢的。
