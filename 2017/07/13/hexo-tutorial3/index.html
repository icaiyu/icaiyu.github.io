使用hexo

前面我们讲完了hexo搭建博客的基本思想。
接下来我们就要谈谈怎么使用hexo，再一次说明，hexo是一款软件，跟你的word，各种单机游戏是一样的。
安装hexo之前，就像你安装各个游戏一样，如果你安装过游戏，你肯定会有这样的经历，比如，游戏安装器会要求你先安装什么visual C++ framework之类乱七八糟的东西，原因就是游戏的运行，有些部件是依靠C++语言的，所以你的电脑上得先有C++。同理，想要在你的电脑上安装使用hexo，那么你得安装nodejs，这是一个计算机编程语言，咱们就别管了，玩了这么多游戏也没见你会去了解C++。
安装如下
先到nodejs官网下载安装包，下载人家推荐的那个，稳定版本好。
https://nodejs.org/zh-cn/
安装的时候记得，选择add to path选项！！！切记
为了确认nodejs成功安装，我们打开cmd黑窗口，输入
node -v
如果能看到
注：你的版本号可能跟我的不一样。
再确认一下nodejs的组件管理器npm有没有随着装好，在黑窗口里面输入
npm -v
如果能看到版本号就说明成功了，如果失败了，十有八九是因为你安装nodejs的时候没有选择add to path。自己重装试试吧。

接下来，我们就需要去寻找hexo的安装包来安装hexo。让人高兴的是，你并不需要像寻找盗版游戏一样满世界去找hexo的安装包，因为nodejs的开发者知道，你安装nodejs十有八九是想要安装依赖nodejs的hexo(当然还有很多其他的依赖nodejs的软件)，所以，nodejs的开发者应经把hexo等这些软件都准备好了，你只要打开CMD黑窗口，输入
npm install -g hexo-cli

nodejs就会自动到准备好hexo的网络仓库那里下载hexo的安装包，然后自动安装。
安装好之后，我们需要把hexo添加到path。
所以要设置一下路径，因为hexo跟一般的软件不一样，它是靠命令行来使用的，我们得在CMD命令行里面输入命令运行hexo，而要想命令行认识hexo指令，我们就得设置环境路径，值得一提的是，很多教程都没有提到这点，后果就是，很有可能你之后运行hexo的指令的时候CMD：说我不懂这个命令。
设置路径的步骤如下：
同时按下ctrl键和E键，打开资源管理器，也叫我的电脑，你用鼠标找到左边的此电脑

按右键唤出菜单，选择属性，你就会见到如下页面，鼠标找到左边的高级系统设置

单击鼠标左键可以看到系统设置的页面，鼠标找到下面一点的环境变量

点击就能看到环境变量的设置，如下图，

我们点击PATH那一行，就能编辑PATH变量了，进入如下图的页面

我们点击新建，
在里面输入
C:\Users\你的计算机名字\AppData\Roaming\npm （注意把你的计算机名字换进去！！！不要傻傻复制啊）
然后一路确认，就可以关闭刚刚打开的各个页面了。这里说一下，我们输入
C:\Users\你的计算机名字\AppData\Roaming\npm
是因为nodejs把hexo装到，这里来了，你需要在资源管理器中（会不会有人不知道啥叫资源管理器？资源管理器就是文件管理器，就是你随便打开一个文件夹，都是打开了资源管理器，看起来是个文件夹，你平时也叫人家文件夹，实际上是一个软件，你可以在这个软件上面的栏目上输入各种路径去进入相应的文件夹），打开看看是不是hexo真的装到这里了，我不保证hexo，hexo一定会安装到这个路径里面。如果有，你会看到如下的东西

总之设置完之后，我们就可以使用hexo，来感受一下hexo的强大威力吧！

hexo可以帮助我们搭建博客，每次搭建博客都是新建一个项目，所以我们先选定一个地方，存放我们生成的文件，我们先在D盘新建一个叫myBlog的文件夹。用来存放我们的项目。
我们打开CMD
输入
d:
先抵达D盘根目录
然后使用指令
cd myBlog
进入myBlog的路径，接着，我们输入
hexo init
稍等片刻就完成了项目的启动
这时还需要运行一下
npm install
这个是安装一些必要的组件用的（第一次使用hexo必须运行这个，第二次的话只要前面的hexo init 就可以了）。

这样我们就生成了一个博客项目，其实这两个命令你并不需要记，除非你会开无数的个人博客。一般而言，一般人只需开一个。所以只会使用一次这两条命令。

我们先来看看这个博客项目包含什么文件。

我之后会逐一讲解这些文件的作用。
我们首先要关注的就是最重要的，也就是，红色所圈着的_config.yml文件，这个文件一看名字就是一个配置文件，我们使用文本编辑软件打开看一下。我建议你使用sublime，用sublime打开myblog文件夹，就可以在sublime的左侧看到myblog的文件结构了，点击_config.yml
就可以查看里面的内容。如下

最上面就是一些关于博客的关键信息，比如，我们博客的标题，也就是我们博客名字，还有作者，都是比较关键的信息，赶紧给你的博客起个名字吧，也把你的网名填到author那一栏去吧。
还有就是language那一行，你可以在后面加上 zh-Hans 。如下
language: zh-Hans
注意，冒号后面有一个空格！！！！！空格！！！！！空格！！！！！
表示把语言设置为中文，如果留空的话，它默认语言是英语。
接下来，我们先运行
hexo generate
或者可以用缩写
hexo g
这条简单的命令你需要记住，以后会经常用。
就可以生成静态文件。
你会发现myBlog文件下里面多了一个public文件，这就相当于我们之前所说的那个www文件夹一样，里面包含的就是博客的主页html，各篇博文的html，这些统统都是所谓的静态文件。
打开public文件夹可以看到里面的东西如下

其中的index.html文件就是主页。

我们运行如下的指令
hexo s
表示开启hexo自带的网络服务器功能，这就相当于，给你的电脑安装了Apache一样，你可以通过浏览器浏览你的博客了。打开一个浏览器，比如chrome，或者edge。输入
127.0.0.1:4000
或者
localhost:4000
你就可以看到你的博客主页了！这真的是历史性的时刻啊，我们看看这上面有什么东西。
