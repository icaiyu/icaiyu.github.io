搞定外观之后，我们需要思考一下内容了。
第一个问题就是，如何为博客添加新的文章？
我们看到新生成的博客里面只有一篇示例的博文，我们如何新增一篇文章呢？
我们知道，无论如何，我们的博客要想让游客通过浏览器浏览我们的博文，那么我们就需要一个HTML文件，里面有我们的文章，还有各种标签来美化我们的页面。难道要手动写HTML文档？这绝对不可能啊。使用hexo这么高端的工具后，我们肯定不会再做那些反人类的活了。比如我们想新添加一篇名为“我的幸福生活”的博文，我们只需要在CMD里面输入
hexo new 我的幸福生活
注意，中文不需要用什么符号引起来！
这时我们就已经添加一篇新的博文了，我们知道，hexo是一个使用命令行的软件，并不像WordPress等东西，有图形界面，不会有什么标题栏这样的东西给我们填入新博文的标题，好吧，我们刚刚输入命令的时候已经输入标题了，那内容呢，有没有文本框给我们输入新博文的内容呢？各位别急，往下看
大家，请打开myBlog文件夹下面的source文件夹，再打开里面的_post文件夹，你会发现，里面有
我的幸福生活.md

我建议你坚持使用，因为markdown已经几乎成为互联网文本的标准了。写markdown的好处是显而易见的，他不需要你像排榜word文件一样动用鼠标来排版，markdown根据规则自动生成的版式非常优美。尤其是编辑数学公式等方面也有很多优势。而且使用markdown并不会像你想象得那么难，像我也并没有刻意去背markdown的规则，需要用的时候百度一下，就好了，甚至我一般也不管什么排版了，文章一股脑写就好了，偶尔关键点用一下markdown的标签就行。
总之，不管你愿意不愿意，如果你想使用hexo，那么你就得接受使用markdown的事实，这也是我们这个文件叫我的幸福生活.md的原因，因为这个就是一个markdown文件啊。
我们继续关注我们的博文。

保存就好了。
输入
hexo g
生成新的public文件夹，里面就是有我的幸福生活.html

这时候开启服务器，
hexo s
浏览博客网站，看到了我们的主页新添加的我的幸福生活的博文，我们可以点击进去查看

这时，我需要解释一下，我们运行
hexo g
到底发生了什么。运行上面程序的一瞬间，hexo会把public删除掉，新建pubic，然后根据source目录下面的_POST中的文件，生成相应的博文HTML放到新的public文件夹里面。
我们可以看到source文件夹的_post文件夹里面有两篇markdown文档，一个是示例的helloWord.md另一个是新添加的我的幸福生活.md，hexo会根据markdown规则，把md文件翻译成博文的HTML文档放进public，而且，博客主页的HTML也会做相应的修改，把新文章做成链接纳进主页去。而且，翻译md文件的时候，并不是简单的翻译，而是要根据主题模板里面的配置进行翻译，比如字体大小，我们可以看到，我们写md文件的时候，里面可是没有说明的字体大小的哦，那么字体多大是由主题配置文件说了算，还有就是，用户浏览到的博文页面的边边角角的内容在md文档里面也没有说明，也都是根据主题配置，和主题文件模板自动生成的。
这就是hexo的开发模式的好处了，该模式下，md文件才是我们最关键的数据，我们只要有md在，随时都能生成相应的public文件夹。不信，你随时可以把public删除，再运行
hexo g
只要source不变，准确说，里面的md内容不变，以及主题配置不变。一切复原。一模一样，也就是说，public完全是依靠主题的模板，和source里面的md文件而存在的，一个负责博客的外观，一个负责博客的内容。所以，你要是对自己的数据非常爱护的话，那么只需要备份md文档就好了，拿出你那块军工级别的三防的移动硬盘，把你新建的md文件备份一下就可以了，顶多再备份一下站点配置文件和主题配置文件如果你不想电脑崩盘重新建立博客的时候再重新配置。你可以看到，需要备份的东西非常少，即使是需要保存的md文件，里面绝大部分也都是文章内容，都是你的血汗，是干货，只有少数字符是markdown的语法符号。只要保有这些文件，你随时就能重建博客，不怕系统崩盘。
说完添加文章，说说如何删除文章，比如我们现在想删除示例的文章，很简单，我们只需删除_post文件夹里面的hello-world.md就可以了，没了这个md文件，你运行
hexo g
新生成的public文件夹里面就不再有对应的hello-world.html
运行
hexo s
博客主页也不再有相关的内容。也就是说，我们成功删除了一篇博文！是不是非常简单！

关于写作，hexo还提供了草稿的功能，输入
hexo new draft 小时代

就会在source中的_draft文件夹中生成一个 小时代.md 文件，你可以在里面写你的小说，这么牛逼的小说，你当然是一时半刻写不完的，所以你这时不希望游客能看到
你运行
hexo g
生成public跟之前是一样的，不会包括你新建 小时代.md，因为hexo知道那是草稿文章，认为你还没写完，所以并不会翻译这篇md成HTML文档。
运行
hexo s
在主页上当然也看不到相应的文章，当然你实在想自己偷偷看看放到网页上的效果如何。
你可以运行
hexo s –draft
你就可以在游览器上看到主页里面包含了你这篇没写完的文章。这只是给你预览的，其他用户是看不到的。
你可以接着在这里面写文章，确认写好之后。运行
publish 小时代.md
就可以把小时代.md从_draft文件夹转移到_post文件夹
运行
hexo g
生成的public就包含了小时代的博文了
运行
hexo s
就能在浏览器上看到了。

所以，hexo的创作功能是非常强大的，只要你习惯了markdown
