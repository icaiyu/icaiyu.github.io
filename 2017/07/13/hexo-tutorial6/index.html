我们之前讲的都是创建博文这一种类型的博客页面，如果我们需要添加别的类型的页面呢，我们看到目前博客网站的页面只有三个，一个是首页，一个是归档，一个是标签，如果我们想添加一个about页面来展示我们的个人信息，那也很简单，只要输入
hexo new page about
这样，我们就创建了一个新的页面了。
你就会发现source文件夹里面出现了一个about文件夹，里面包含的就是用来翻译成about.html的
index.md
我们在元信息里面添加入这个页面的类型
type: “about”
注意冒号后面的空格，然后在原信息下面填入你的个人个性说明
接下来，如何让它出现在右上角的菜单栏里面呢？

我们打开主题配置文件进行配置，在45行左右，你将找到负责配置菜单栏的那一部分配置代码。
你可以看到，里面没有被注释的配置都已经出现在网页的菜单栏上面了，所以，我希望about页面出现，那么，我们就把about那一行配置前面的井号去掉让这个配置有效就可以了，

你还可以调整这些配置的位置来调整他们出现在菜单中的位置，比如，你把关于about的配置剪切，复制到后面，那么about就会出现在菜单栏的最右边。

你看这些配置中有一个categories是无效的，也就是说，目前你的博客网页还没有开启分类的功能，如果，开启了，会怎么样呢？
首先，我们按照之前新建about页面一样，新建一个用来划分的类别的页面
hexo new page categories
你就会发现在source文件夹里面出现了categories文件夹，里面就是用来翻译成categories.html的markdown文件，我们需要在元信息里面添加该页面的类型声明。添加如下代码
type: “categories”

然后在主题配置文件的菜单那里把categories那一行前面的井号去掉。这时我们就可以查看我们的分类页面了。
在此之前，我们还需要对文章的类别进行声明，比如，我之前的那篇 我的幸福生活 的博文，我们打开source文件夹中的 我的幸福生活.md 我们看到它的元信息里面有title，有data，有tags，我们在后面插入一行
categories: 生活随笔

我们就这样把这篇文章归到生活随笔这个类别之中去了。浏览一下试试吧

接下来，你可以使用
hexo new 我爱吃的餐馆
然后你，在这篇新文章的元信息里面添加
categories: 生活随笔
它就会被自动归类到 生活随笔 那一类里面。
如果你添加的是
categories: 吃货笔记
那么hexo就把它归为新的一类，所以归类是非常方便的。
这时你就会想了，如果，我最近写的文章都是固定的一类的，那么有没有什么办法在新建文章的时候就出现默认的类别呢，这样就不用手动输入类别了。比如我最近在写hexo教程，最近新建的每一篇文章都是属于hexo教程这一类的，那么我就打开source旁边的scaffolds文件夹，你看到里面有三个md文件，我打开其中的post.md

我在元信息里面填好
categories: hexo教程
那么，以后我每次使用
hexo new 不管是啥标题
生成的 不管是啥标题.md 里面最开始就包含了
categories: hexo教程
这条元信息，实际上，你会发现
不管是啥标题.md 在生成的时候，里面的内容跟scaffolds里面的post.md的内容一模一样。我们除了可以设置元信息，还可以写一些会反复出现的内容，比如，每篇教程的前面，我都想加入一句问候的话，“大家好啊，我终于又推出一篇新文章了，大家有没有迫不及待地想读呢”之类的话语。这样就不用在每一篇文章里面手动添加了。
这就是scaffolds存在的意义了，他就是一个“脚手架”用来帮助我们搭建文章用的。同理，scaffolds文件夹里面的draft.md就是帮助我们生成草稿用的，page.md就是帮助我们生成页面用的，这些都可以随意定制。是不是感受到hexo的强大了？
值得一提的是，每一篇论文最多属于一个类别，但是，每篇文章却可以贴上很多tag，也就是贴上很多标签，比如我的这篇文章，类别是hexo教程，那么我可以贴上“技术文”，“干货”，“吸粉文”之类的标签。至于具体怎么弄标签功能，就留给读者自己探索吧。只要敢于尝试，hexo总能给你惊喜。
我们的hexo使用教程到这里就告一段落了。希望你们玩的开心。之后，我们就要开始博客的部署了！
