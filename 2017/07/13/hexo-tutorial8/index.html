设置完io仓库，就下来就简单了，我们只需要把public文件夹里面的文件复制到仓库里面就好了。如果你使用过百度云作为存储文件的仓库，那你一定觉得GitHub跟百度云一样肯定有什么按钮是用来上传文件的，然而，你很快就会发现GitHub只有上传文件却没有上传文件夹的功能，GitHub作为程序员专属的仓库，如果还需要通过那么低级的手段上传代码，那岂不是很没面子，所以程序员都是通过专属工具来上传文件的，这个工具就是git。上传文件只是git功能的一个小部分，git主要是一个版本控制器，这里简单科普一下啥叫版本控制器，如果你写过论文，那么你的文件夹里面有很大概率会出现如下的文件
《论文最终版》
《论文最终版2》
《论文绝对不修改版》
《论文绝对绝对不修改版》
《论文打死都不改版》
《论文添加最终数据版》
《论文被老板骂了之后的修改版》
《论文。。。弃疗版》
论文需要多次修修改改，有些修改你又不确定是不是真的要把某部分删掉，也不确定删掉部分将来会不会反悔想加回来，所以你不敢随便删除，所以每次修改，都另存为，于是每次都增加一个文件。最终导致文件夹里面充满文件，然后你也不知道，哪个里面有哪些独特的内容，最后崩盘。
实际上，程序员写代码也会遇上这些情况，但是，使用git就能解决这些问题。所以，git是程序员们最最最最为常用的工具之一。如果，你也被各种论文版本所困扰，不妨去廖雪峰老师那里学习一下
我们从GitHub的名字就能看出，GitHub跟git有千丝万缕的关系，要想自动往GitHub传送文件，我们电脑肯定是要安装git的，git跟hexo一样也是软件，也是命令行软件（git也有图形界面，不过非常非常不建议使用）
到git工具的官网下载页面，点击windows图标，
https://git-scm.com/downloads
然后选择对应的git就能下载windows版本的git

如果选择你的电脑相应的位数的git
如果你不知道你的电脑啥位数的，选择
32-bit Git for Windows Setup
然后一路安装就好了，里面关键的配置，人家都给你默认选好了，咱们身为小白就不要随便修改了。反正一路next到安装就好了。

装完之后，我们需要启动git。
按下电脑的菜单键，也就是有微软logo的按键，呼出菜单，找到git

，打开git bash
就启动了CMD命令框，虽然跟我们之前的CMD命令框长得不太一样，但是实际上是一个东西，只不过，会提前把git工具，以及git需要的其他组件启动好了，我们可以使用git工具了。

git本身是一个版本控制器，它是不能实现文件传输的功能的，但是安装git的时候自动装了一个叫ssh的组件，它会负责文件在网络之间的传输，这就是我们需要启动git bash的原因，启动这个的时候会自动启动ssh为文件传送做好准备。
我们传输文件的时候会输入目的仓库的名字，所以GitHub会把文件送到我们对应的仓库里面，而不是送到张三那里，可是远方的GitHub仓库怎么知道我们的身份，允许我们往仓库里传文件呢，所以，往我们的GitHub仓库传输文件之前，我们必须设置身份文件，也就是RSA，传说中的公钥和私钥身份验证器。
这种方法如何验证身份呢。第一次传输文件之前，git里面集成的ssh工具，会根据我们的计算机自动生成一个私钥文件和一个公钥文件（这些文件是一堆乱码，但是是独一无二的，是根据我们使用的计算机信息计算出来的），其中的公钥就相当于钥匙，我们可以随便配钥匙，随便发给任何人，这里，我们会把公钥里面的乱码复制一下粘贴到GitHub的配置里面去，相当于，我们配了一把公钥钥匙放到GitHub那里，每次我们每次使用ssh协议传送文件的时候，就会拿着私钥，准确的说应该叫私锁，相当于一把私密的锁，去跟GitHub账户中的公钥配对，因为我们之前在GitHub账户里面放了公钥，私锁和公钥能配得上，也就是说，我们配给GitHub的公钥能打开我们这把私锁，于是，GitHub就认定，我就是当时存放公钥的人，于是允许我往上面传文件。我们说了，公钥随便发，最好什么人都拿着，反正他们不能够根据钥匙造出锁来，他们拿着钥匙来我们的仓库是没啥用的，我们的仓库不认，钥匙谁都可以有，你只要拥有一把锁，而这把锁能用我这的钥匙打开，我们才给你放行。这就是RSA身份认证的原理。
这种思想，我国古代就有了，古代调兵的虎符就是这样的原理，皇帝手里拿着虎符的左半部分，天下只有这么一个左半部分，手下的将军们，手上拿着的东西都是一模一样的，都是虎符的右半部分，右半部分随便复制给将军们随便发，甚至士兵也可以人手一个，反正，我们有技术可以保证单从右半部分不能制造出左半部分。皇帝想调遣骠骑将军去攻打匈奴，去宣旨的太监只要亮出左虎符与骠骑将军匹配一下，左右合璧，骠骑将军就知道，这确实是皇帝的命令，于是出兵。而某个车骑将军想谋反，想假借皇帝命令将骠骑将军调走，但是他手里拥有的也还是右边的虎符，肯定骗不了骠骑将军。

所以，我们第一步就要使用ssh命令为我们的电脑生成公钥和私钥，输入
ssh-keygen
意思是ssh key generate
这时应该在你的电脑的C盘用户目录，你的用户名目录下面有一个叫.ssh(注意该文件夹前面有一个点)

里面会有两个这样的文件

其中id_rsa就是私钥文件，而id_rsa.pub就是公钥文件。
我们使用sublime把公钥打开，里面有一堆乱码，全选，复制
然后到你的GitHub账号页面，点击右上角的图标，点击setting

然后设置页面的左边选择SSH and GPG keys

然后点击上面的 New SSH key

然后在Title里面随便填点东西，比如“我的笔记本的公钥”
在Key里面黏贴之前复制的公钥乱码内容
最后是点击Add SSH key
以后，我们就可以在自己的电脑上通过ssh来往仓库传输文件了。如果你是和你的好基友一起维护的博客，你可以叫你的好基友按照上面的办法安装git，然后在他电脑上生成ssh钥匙，然后把他电脑的公钥也添加上去，这样你的基友也可以往该仓库传送文件了。
然后就可以，关闭git bash 回到我们之前使用hexo的原始CMD来了。因为我们并不会直接使用git。我们可以通过配置，让hexo自己调用我们电脑上的git来往仓库传输文件。
配置这么重要的东西，当然需要在我们的hexo的站点配置文件里面进行，你在该配置文件的最下面就能看到deploy字眼，那就是专门用来配置部署博客用的。
配置如下

记得把里面的username改成你的GitHub用户名
配置好之后，我们先运行
npm install hexo-deployer-git –save
注意：save前面是两个-
这是安装hexo需要的部署工具
然后运行
hexo d -g
意思，先generate，再deploy。我们先在本地生成public文件夹，然后往GitHub仓库推送public文件夹里面的文件。
以后，你对文章做完修改，或者写了新文章，只要打开CMD黑窗口，抵达myblog目录，运行
hexo d -g
就可以自动生成新的public，自动往仓库传输public文件。别人就能够通过
username.github.io
来访问你的博客了，如果你对这个域名不满意你也可以自己购买域名（比如我买的域名就是lier.space这比lier2017.github.io好多了，因为lier.github.io已经被抢走了），然后在GitHub上把新域名很这个仓库绑定，以后别人就可以通过你购买的域名来访问你的博客了，这些步骤不在本教程的范围之内。

至此，我们完成了一个博客的搭建，一个真正的，能够投入使用的博客。接下来，我们要把精力放到写文章上来，毕竟颜值和技术都是浮云，内容才是吸粉最关键的东西，好好写作吧少年！
