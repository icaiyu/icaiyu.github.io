说明:本人最近翻译sqlite3的python API文档。该文档较为详细地介绍了sqlite3的特性，但是，我认为对于初学者来说，很多内容其实都用不到，因此，我在此写一个较为简短的教程，总结主要的内容，以便初学者能够快速上手。

sqlite3 简介

sqlite3其实就是一个轻量化的sql数据库，类似mysql，如果你对mysql不熟悉，也无关紧要，事实上，几乎所有的关系型数据库设计概念都跟Excel相同（这个你总归是熟悉的吧）。
一个sqlite3的数据库，就是一个”xxxx.db”其实就对应一个”xxxx.xls” ，在一个数据库里面我们又可以有不同的表格，正如xls里面会有很多的sheet。不同的数据存到不同的表格里面去就可以了。其实原理是一样的，只不过微软把Excel做成了图形界面，而sqlite3需要指令去完成建库，建表，填写，修改，删除，等工作。事实上，sqlite3也有图形界面让你操作sqlite3就像操作Excel一样简单。但是，sqlite3提供了python-api接口，所以，我们可以在python程序里面用python 代码去实现操作。 这里，我假设你对python有一定的了解。

模块关键方法 sqlite3.connect()

sqlite3.connect(database[, timeout, detect_types, isolation_level, check_same_thread, factory, cached_statements, uri])

这几乎是sqlite3最基本的方法了，所有的操作从这里开始。

Connection 对象

这几乎是sqlite3最基本的对象，每次操作一定要有。
下面介绍这个对象有哪些关键的方法和属性

cursor(factory=Cursor)

该方法生成一Cursor对象，这个Cursor对象将负责将来的数据查询，修改等等工作，该方法还支持传入一个factory参数，如果你对原生的Cursor不满意，你可以自己设计一个Cursor子类，然后设计一个工厂函数来生产这个Cursor子类。

commit()

此方法提交当前事务。 如果您不调用此方法,则自从上次调用commit()以来执行的任何操作都不可见于其他数据库连接。 如果您想知道为什么您没有看到您写入数据库的数据,请检查您是否忘记调用此方法。

rollback()

此方法回滚自上次调用commit()以来对数据库的所有更改。滚回上次提交前。

close()

这将关闭数据库连接。 请注意,这不会自动调用commit()。 如果您只关闭数据库连接而不先调用commit(),则更改将会丢失！

execute(sql[, parameters])

这是一个非标准的快捷方式,一般来说，是由Cursor对象使用execute方法，如果你直接让Connection对象使用该方法，它背地里通过调用cursor()方法创建一个Cursor对象,使用给定参数调用Cursor的execute()方法,并返回Cursor。

executemany(sql[, parameters])

这是一个非标准的快捷方式,一般来说，是由Cursor对象使用executemany方法，如果你直接让Connection对象使用该方法，它背地里通过调用cursor()方法创建一个Cursor对象,使用给定参数调用Cursor的executemany()方法,并返回Cursor。

executescript(sql_script)

这是一个非标准的快捷方式,一般来说，是由Cursor对象使用executescript方法，如果你直接让Connection对象使用该方法，它背地里通过调用cursor()方法创建一个Cursor对象,使用给定参数调用Cursor的executescript()方法,并返回Cursor。

create_function(name, num_params, func)

创建一个用户定义的函数,您可以在SQL语句中通过使用name的方式调用该函数。 num_params是函数接受的参数的数量(如果num_params是-1,函数可以接受任意数量的参数),func是一个可调用的Python,它被称为SQL函数。该函数可以返回SQLite支持的任何类型:bytes,str,int,float和None。

从上面的内容，我们可以看到，只要一个Connection对象，就能轻松玩转数据库了，因为它自带了Cursor的操作的快捷方式。

Cursor 对象

一个Cursor有下面的属性和方法.

execute(sql[, parameters])

执行一条SQL语句。 SQL语句可以被参数化(即占位符而不是SQL文字)。 sqlite3模块支持两种占位符:问号(qmark样式)和命名占位符(命名样式)。

executemany(sql, seq_of_parameters)

seq_of_parameters是一个序列(比如list,tuple等)，对seq_of_parameters序列的每个元素都执行以下sql语句。 sqlite3模块还允许使用迭代器产生参数而不仅仅是序列。

executescript(sql_script)

这是一次执行多个SQL语句的非标准便利方法。 它首先发出COMMIT语句,然后执行SQL脚本。

注意： 请仔细区别 executemany 和 executescript

fetchone()

获取查询结果集的下一行,返回单个序列,或在没有更多数据可用时返回None。

fetchmany(size=cursor.arraysize)

获取查询结果的下一组行,并返回一个列表。 没有更多行可用时返回空列表。

每次调用要获取的行数由size参数指定。如果没有给出,Cursor的数组大小决定了要获取的行数。该方法应该尽量按照参数给定的大小读数据。如果没办法指定具体的size,则可能返回的行会比较少。
请注意,大小参数涉及性能方面的考虑因素。为了获得最佳性能,通常最好使用arraysize属性。然后如果使用size参数,那么最好fetchmany()调用保持相同的值。

fetchall()

获取查询结果的所有(剩余)行,并返回一个列表。请注意,Cursor的arraysize属性可能会影响此操作的性能。没有行时返回空列表。

close()

现在关闭Cursor。从这一时刻开始,Cursor对象将无法使用; 如果用Cursor尝试进行任何操作,将会引发一个ProgrammingError异常。

Row Objects

Row实例充当Connection对象的高度优化的row_factory。设定好之后，查询操作返回的每一行数据会被包装成Row对象,而这里的Row对象通过优化设计，使得他们具有很好的实用性，甚至可以代替fetchall()等方法

它支持按列名和索引,迭代,表示,相等性测试和len()等属性进行访问。 如果两个Row对象具有完全相同的列并且它们的成员相等,则它们相等。

keys()

此方法返回数据库的表格的列的名称的列表。 在查询之后,它立即成为Cursor.description中每个元组的第一个成员。

SQLite和python之间的数据转换

SQLite本身支持以下类型:NULL,INTEGER,REAL,TEXT,BLOB。

因此可以将以下Python类型发送给SQLite,而不会有任何问题:

Python type SQLite type
None NULL
int INTEGER
float REAL
str TEXT
bytes BLOB

这是默认情况下SQLite类型转换为Python类型的方式:

SQLite type Python type
NULL None
INTEGER int
REAL float
TEXT depends on text_factory, str by default
BLOB bytes

对于复杂的python类型,我们可以通过adapter函数将负责的python类型转为sqlite能识别的的基本类，然后读数据库的时候又通过converter函数将基本类转为复杂的python类型
