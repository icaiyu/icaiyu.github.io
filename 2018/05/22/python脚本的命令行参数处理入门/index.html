前言

最近在帮忙部署一个网站,需要抓取github上别人提交的markdown文件,比如一些公告,新闻之类的,然后渲染成html文件放到服务器上,所以需要些一点简单的脚本完成这项运维工作(实际上这事就相当于一个极简版的hexo,转md为html然后自动部署),由于没有时间学习bash脚本语法了,而python也强大到足以胜任这样简单的任务,此外,python由于其简单易懂的语法,大大减轻了书写脚本的难度,这也是为什么python被视为脚本语言的原因。

书写python最为关键的一步是解析参数,因为,我们想要自动完成的任务可能是错综复杂,可能是一个包含大量小任务的大任务,因此,我们我们的脚本必须能够获取的简单的参数,进行相关的任务。比如,我们要做的工作有 task1 和 task2。我们每次运行脚本不一定要每个任务都做一遍,有些任务可能需要一天运行一次,有些任务,可能一个月才一次。比如每个月抓取 当月报表.md 转成 当月报表.html 添加到主页, 然后每天可能都有新的公告出炉,我需要每天抓取一下公告的文件夹,把里面的md文件全部转成html添加到公告页。所以我需要这样运行脚本

位置参数和可选参数

看下面的例子感受一下这两个参数的区别,我假设你会一点linux的操作

我们可以看到 ls 是一个linux指令,能够列出某目录下的全部路径和文件,它需要配合一些参数进行使用。
如 ls pypy 意思是查看 pypy目录里面的文件,
我们还可以 ls pypy cpython 列出 pypy 和 cpython 目录下的全部文件
这里的 pypy 和 cpython 就是位置参数。
而 ls -l 中的 -l 就是可选参数,这个参数表示把文件的详细信息列出来。

  1. 在没有任何选项运行时,ls命令很有用。它默认显示当前目录的内容。
  2. 如果我们想列出其他的我们想要的内容,我们会多说一点。比如,我们希望它显示一个目录pypy的内容。我们所做的是指定所谓的位置参数。之所以叫位置参数是因为程序仅仅根据它在命令行上的显示所在的位置就知道如何处理该值。这里,对于ls目录来说位置或许不重要,但对于cp这样的命令,就很重要了,cp最基本的用法是cp SRC DEST。第一个位置是您想要复制的位置,第二个位置是您希望复制到的位置。
  3. 现在,假设我们想显示每个文件的详细信息,而不是仅显示文件名。在这种情况下,-l被称为可选参数。
  4. 此外,你的脚本一本都要准备一个help文本,告诉别人使用你的命令行或者脚本

参数进阶版的参数解析 Argparse

argparse 是python内置的命令行参数解析模块，专门用以解析命令行参数，如果你阅读比较旧的代码，还有可能看到 optparse 这是python曾经内置的命令行解析模块，argparse 是它的继任者，不计继承了optparse的功能还变得更加强大。
